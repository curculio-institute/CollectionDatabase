Collection Database — Schema Reference

Columns prefixed dwc: are Darwin Core terms and link to dwc.tdwg.org/list. Local columns (PKs, FKs, management fields) carry no prefix. Core/label tables use SQLite STRICT mode (see Integrity Constraints). Note: the per-column listings below reflect migration 0012 and are partly stale (some columns since renamed/dropped); the Integrity Constraints section reflects the current intended (post-0029) schema.

Table Reference

taxon

Local taxon / OTU. DwC parent-link model (GBIF checklist best practices). Hierarchy encoded via dwc:parentNameUsageID (FK to self) — no denormalised rank columns. Each ancestor rank (genus, family, order, …) has its own row linked upward by the same FK. name_element is the atomic source of truth (this rank's own epithet/uninomial, e.g. crypticus); dwc:scientificName stores the composed full name without authorship (e.g. Otiorhynchus crypticus), maintained from name_element + the parent chain (migration 0032, Epic #30). Synonymy is encoded solely by dwc:acceptedNameUsageID: NULL = accepted, set = synonym pointing to its accepted taxon (the DwC taxonomicStatus term is derived at export, not stored — dropped in migration 0030). Each name carries its own lineage, so a synonym keeps its own parentNameUsageID (its own genus) independent of its accepted name (Epic #30). The one surviving rule: the accepted name must itself be accepted, not another synonym — a BEFORE trigger (migration 0031); the former synonym-parent-match trigger was retired in migration 0033. See Integrity Constraints → Triggers.

Integrity Constraints

STRICT typing

Core, label, and print-queue tables are declared SQLite STRICT, so column types are enforced at write time (no silent affinity coercion). Exceptions: person and person_defaults are not STRICT.
Note: migrations 0020/0021/0024–0027 recreated several tables via batch_alter_table(recreate=…) and dropped STRICT + CHECKs; migration 0029 restores them. This page documents the intended (post-0029) schema.

Triggers

Cross-row rules a CHECK cannot express (it sees only one row). These BEFORE triggers RAISE(ABORT) on any violating write — raw SQL included. Added in migration 0031; the synonym-parent-match trigger was retired in migration 0033 when the model moved to own-lineage parenting (Epic #30). A taxon table rebuild silently drops triggers (like it drops CHECKs/FKs — see STRICT note); any rebuild migration must re-create them, and tests/test_schema_integrity.py::test_synonym_integrity_triggers_present guards it. The drift a write-time trigger cannot catch (e.g. a raw-SQL edit that chains a synonym after the fact) is surfaced by the manual verify_taxon_consistency() audit (Taxonomy-tab "Check consistency").
